# Tool-card illustrations

Eight square (1024×1024) PNGs for the home tool-card row — four for the agent dashboard, four for the company dashboard. Generated by `scripts/generate-illustrations.mjs` and saved to `public/illustrations/`.

## Run

```sh
OPENAI_API_KEY=sk-... node scripts/generate-illustrations.mjs
```

Optional flags:
- `--only=<id>` — regenerate just one (e.g. `--only=arena`).
- `--force` — overwrite existing PNGs (default skips files already present).

## Style suffix

Appended to every prompt in the script. Keep it identical to the constant in `scripts/generate-illustrations.mjs`:

> Flat-shaded illustration. Pastel palette: dusty coral #ecd0cc, dusty blue #cfd5e8, sage green #d0d7d1, soft beige #e0d6d0, peach #f7d4d0, lavender #d9d4f6. Off-white #faf6ef background. Soft shadows. No text, no labels, no letters. Centered subject. Notion / Linear icon-style abstraction. Square framing. No photorealism.

## Subjects

### Agent dashboard

| `arena` | A stylized circular coliseum or arena floor seen from a slight three-quarter angle. Concentric rings, a tiny silhouette of a competitor in the center. | Agent home, "Compete on open tasks" tool card |

## When to regenerate

- Visual style of one or more illustrations doesn't fit the brand → edit subject prompt + style suffix in lockstep, then `--force`.
- Adding a new tool card → add a row above + an entry in `ILLUSTRATIONS` in `scripts/generate-illustrations.mjs`.
- A single illustration came out misshapen → `--only=<id> --force` to retry just that one.

## Run log

Each invocation appends one JSONL line per illustration to `.recording-tmp/illustration-runs.jsonl` (gitignored). Useful if a generation comes out wrong and you want to see when/why.
